# Enterprise Portfolio Architecture - Omni-Dispatch Setup Runbook

This document outlines the setup parameters for the **Omni-Dispatch Architecture**. The system utilizes a Vite/React Frontend coupled tightly with a true asynchronous Python FastAPI backend handling MongoDB persistence, Telegram pushes, and SMTP formal triggers seamlessly in parallel.

---

## 1. System Architecture Audit

- **Frontend (`/src`)**: A high-performance React SPA built with Vite. Responsive and styled natively using HSL tokens.
- **Backend (`/backend`)**: A FastAPI microservice adopting Domain-Driven structure. Handles telemetry HTTP request proxies and rate limiting logic.
- **Primary Database (MongoDB)**: Forms are permanently captured and stored in `portfolio_db.contact_messages` utilizing the `motor` asynchronous driver via FastAPI lifespan injection.
- **Notification Transit (Omni-Dispatch)**: Rather than using 3rd-party SaaS integrations, the server natively constructs standard HTML `httpx` arrays routing to your private Telegram Bot, racing concurrently alongside Native TLS SMTP integrations.
- **Cache Layer**: Redis implements a fail-open token-bucket constraint.

---

## 2. Infrastructure Setup (Local Development)

### Prerequisites
- **Node.js**: v18+ 
- **Python**: 3.12 (specifically tested via `pyenv`)
- **Redis Server**: Local `redis-server` running on `6379`.
- **MongoDB**: Local `mongod` cluster running on `27017` (or access to an Atlas Cloud URI).

### Backend Engineering Stack
1. Crack open a terminal and shift to the backend slice:
   ```bash
   cd backend
   ```
2. Install the native dependency stack (includes `motor` and `aiosmtplib`):
   ```bash
   pip install -r requirements/base.txt
   ```
3. Initialize the Omni-Dispatch configurations. Verify `backend/.env` exists and fill it comprehensively:
   ```env
   # Database (The source of truth)
   MONGODB_URL=mongodb://localhost:27017
   
   # Telegram Notify Pipeline
   # Get these by Messaging @BotFather and your @RawDataBot on Telegram
   TELEGRAM_BOT_TOKEN=your_bot_token_here
   TELEGRAM_CHAT_ID=your_chat_id_here
   
   # SMTP Fallback Pipeline
   # Requires a Google App Password (2FA -> App Passwords)
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=465
   SMTP_USER=nyangaufredrick443@gmail.com
   SMTP_PASSWORD=your_google_app_password
   ```
4. Boot the Microservice stack:
   ```bash
   uvicorn app.main:app --reload --port 8081
   ```

### Frontend Interceptor
1. Open a secondary terminal.
2. Ensure you have a standard `.env` running in the root `/home/ubuntu/Projects/portfolio(old)/` containing:
   ```env
   VITE_API_URL=http://127.0.0.1:8081
   ```
3. Boot the pipeline:
   ```bash
   npm run dev
   ```

## 3. Production Deployment Manual

### Backend Deployment (Render / AWS)
1. Ensure your Render web-service points firmly to the `./backend` **Root Directory**.
2. **Build Command**: 
   ```bash
   pip install -r requirements/base.txt
   ```
3. **Start Command**:
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port $PORT
   ```
4. **Environment Variables Check**: Ensure your Cloud MongoDB URL, Telegram Bot Token, Chat ID, and App Password are fundamentally stored in Render Env Vars.

### Frontend Deployment (Vercel)
1. Add the Vite SPA root to Vercel.
2. Under strictly **Environment Variables**, lock your `VITE_API_URL` exclusively to the Render Cloud address (e.g., `https://portfolio-api.onrender.com`).
3. Deploy.

## 4. Troubleshooting the Pipeline

* **422 Unprocessable Entity generated by Frontend?** 
  Your submission payload failed Pydantic Verification strictly before hitting MongoDB. Ensure your test message meets the minimum length condition dynamically (e.g., > 10 characters).
* **Telegram Network Degraded?**
  FastAPI organically traps the TLS/HTTP error arrays. SMTP will deploy successfully anyway alongside permanent MongoDB commit statuses regardless. Check logs for the exact array response.
* **Timeout fetching Live Stats?**
  The proxy catching downstream renders failed mapping properly. The frontend catches gracefully rendering default cached displays seamlessly!
